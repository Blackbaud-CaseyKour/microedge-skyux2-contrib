This list view grid component displays a SKY UX-themed grid of data. It includes the ability to defined a custom search function for the data given. Ideally it is used in conjunction with the list component. Specifies the name of the view. Acceptable values: string. Specifies the columns to be hidden by default. Will match off of the id or field of the item. Acceptable values: Array of strings or Observable Array of strings. Specifies columns to be displayed by default. Will match off of the id or field of the item. Acceptable values: Array of strings or Observable Array of strings. Specifies how the gird should fit to its parent. Acceptable values: 'width' or 'scroll'. Specifies the height of the grid. Acceptable values: number or Observable number. Specifies the width of the grid. Acceptable values: number or Observable number. Specify a search function to apply on the view data. Acceptable value: (data: any, searchText: string) => boolean. Specify if the grid will have a static selector column with checkboxes. Acceptable value: boolean or Observable boolean. Specify a unique identifier for the column. Is required if field is not specified. Specify rows to render for a given column from data passed into parent component. The specified field must match the attribute of the item in the data array. For example, given a data array of items: [{{ '{' }} id: 1, column1: apples {{ '}' }}, {{ '{' }} id:2, column1: bananas {{ '}' }}], specify column1 as the field to map all items as rows to the designated column. Fields also enable the column to be sortable, searchable, and selectable to be displayed or hidden. Is required if id is not specified. Display text for the column header. Specify a search function for a given column. By default a string comparison search function is applied on the column data. Specify a column width. Boolean determines if the column is draggable. Boolean determines if the column is hidden. Specify HTML template to render for each row for a given column.
